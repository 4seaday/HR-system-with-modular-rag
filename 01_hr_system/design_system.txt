<인사시스템 설계>

1. 직원 조회 기능
    - 단순 검색 이상의 것
    - SQL 또는 검색 기능
    - 예를 들어, "김패캠" 이름을 조회한다고 하는 건 RAG 필요 X
    - 자연어 질의에 대해서, 실제 데이터베이스에 있는 내용을 기반으로 답변하는 것을 원함
    - "백엔드 잘하는 직원 보통 어느 부서에 있지?"
    - "요즘 직원들의 관심사 뭐가 있을까?"
    - SQL 생성 기능
    - 프롬프트로 현재 데이터베이스 구조를 전달해야함

2. 자기소개서 평가 기능
    - 여러 기능을 수행할 수 있어야 함
    - 기능을 수행하려면 task에 적합한 프롬프트가 전달되어야 함
    - 우리가 수행할 task에 적합한 프롬프트를 미리 만들어 놓음
    - 기능 요청 -> 적합한 프롬프트를 검색할 수 있어야 함
    - 프롬프트 검색 기능 추가
    - 적합한 프롬프트로 실제 기능을 수행할 수 있어야 함
